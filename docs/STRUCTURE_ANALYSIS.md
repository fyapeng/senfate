# Natal structure analysis

## Boundary

`analyzeNatalStructure` consumes only a certified four-pillar state and a
validated model profile. It does not read civil time, location data, raw source
records or topic rules.

```text
FourPillarState × SenFateModelProfile
→ pillar ontology expansion
→ finite element measure
→ root and exposure snapshot
→ day-master support-pressure functional
→ natal dynamic state
→ weighted relation competition
→ stable ReferenceNormalFormPhaseResult
```

## Ontology expansion

Every visible stem is mapped to element, polarity and its ten-god relation to
the day stem. Every branch expands through the canonical main, middle and
residual hidden stems, and each hidden stem receives the same explicit mapping.

## Element and strength functionals

The element measure sums visible stems and hidden stems using profile position
and hidden-rank coefficients, then applies the month-branch seasonal multiplier.
The result is a finite positive measure over `{木, 火, 土, 金, 水}`. It is a model
quantity and must not be described as an observed natural proportion.

Day-master evaluation partitions the measure into same-element, resource,
output, wealth and officer components, with separately recorded root mass. The
support ratio is classified only through the selected profile thresholds.

## Relation normal form

Detected combine, clash, harm, break, punishment, three-harmony and
three-meeting candidates enter weighted competition. Each relation receives an
explicit disposition: `effective`, `transformed`, `contested` or `blocked`.
Downstream layers may consume only the stable result and its fingerprint.

Pattern, climate and balancing now exist as parameterized projections over the
stable result. They are labelled as candidates and coordinates rather than
cross-school conclusions. Kinship, topic and event output still require their
separate reference-rule audits.
