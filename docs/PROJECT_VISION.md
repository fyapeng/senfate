# SenFate 产品定义

SenFate 要实现一套可追溯、可检验、可逐层解释的现代八字推断系统。古籍原文提供规则证据，现代形式系统负责定义输入、推断对象、依赖关系、冲突处理和输出边界。

系统不允许从某个干支字面直接跳到人生事件。任何结论必须经过五行与阴阳、藏干、十神、生克、结构测度、关系裁决、正规形、六亲或主题语义等中间对象。缺少中间语义时，规则进入明确的延期队列。

产品分为三个边界清楚的部分：

- 计算内核：纯函数、确定性输入、版本化模型和完整证书；
- API：运行计算并返回公开合同，不托管网页；
- Web：解释输入、过程、证据和结果，不复制领域逻辑。

前端采用 Astro 静态页面和必要的 React 交互组件，由 GitHub Actions 构建并发布到 GitHub Pages。Cloudflare Worker 只拥有 API 路径。

## 网站信息架构

第一层导航固定为四页：

- 排盘分析：出生时空输入与命盘、结构、生命周期、主题、依据五层工作台；
- 推断原理：历法口径、关系图、正规形、动态生命周期与主题证书；
- 现代视角：历史知识系统、形式化能力边界与经验验证议程；
- 模型配置：预设选择、参数分类、敏感性与版本化导出入口。

界面设计采用现代、克制、研究型的产品语言。总纲用于约束产品宗旨、信息架构和推断边界，不作为逐像素 UI 设计稿。

在正式 Calendar Engine 与正规形主题运行时接通前，排盘结果和模型参数区必须明确标记为产品演示，计算按钮保持失败关闭。演示值不得与用户填写的出生信息建立伪计算关系。
